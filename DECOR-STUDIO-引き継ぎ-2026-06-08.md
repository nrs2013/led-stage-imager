## このアプリは何？

コンサートのLED画面に映る「電飾」（線・電球・三角など任意の図形）を、**本物のDMX卓**から操作して光らせるMacアプリ。

- 卓の信号は **Art-Net（LANケーブル）** でMacに入れる
- 描いた図形に **番地（DMXアドレス）** を割り当てて、RGBで色・明るさを操作
- 電飾**以外は真っ黒**にして、**Syphon** で **Resolume Arena** に渡す（黒は「加算」で消えるので、電飾だけが本番映像に乗る）
- **パワポ感覚**で、チャート画像を下絵に敷いて、その上に図形を描いていくのが中心の作業

## 合意済みの大事な決定（ここは変えない前提）

1. **DMX入力 = Art-Net**（卓とMacを同じLANに繋ぐ）。
2. **出口 = Syphon で Resolume へ**（同じMacで動かす前提）。もしResolumeが別PCなら **NDI** に差し替え。
3. **番地は既定で3チャンネル（R / G / B）**。
   - **バーチャルディマーは卓側で処理**（卓が明るさをRGBに掛けてから送る）→ アプリ側はディマー用の番地を使わない。
   - 4ch（RGB+Dim）も選べるが、**既定は3ch**。
4. 描いた**図形そのものが光る**（線で光る／塗りで光る、図形ごとに選択）。
5. **同じ番地を複数図形に付ければ一斉点灯**。別番地なら個別。
6. **画面サイズは自由**（〜4K、4K超えはほぼ無し）。チャートは**JSON1ファイル**で保存（下絵画像も埋め込み）。
7. 初版は**ピクセルマッピングなし**（1図形＝1色。線の途中で色を変えるのは次版送り）。

- **0.【最初の難所】実証** — 偽の卓（テスト送信）→受信→黒背景に四角を1個光らせる→Resolumeに出す。**ここが光って動けばGO**（他を作る前にこれを必ず先に通す）

- **3. 番地割り当て＆一覧表**

## 最初の難所（段階0）で見るポイント

- 動かし方（プラン Task 0.4〜0.5 に詳細）：
  - アプリ起動：`npm run dev`
  - 偽の卓：`node tools/artnet-test-sender.mjs 0`
  - Resolume を開いて Syphon/Spout ソースに「**DECOR STUDIO**」が出て、四角の色が偽の卓に合わせて脈打てばGO。
