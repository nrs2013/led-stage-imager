# DECOR STUDIO 設計書

**日付:** 2026-06-08
**仮称:** DECOR STUDIO（電飾を DMX で光らせる Mac アプリ）
**作る人:** のむさん（コンサート演出家・GitHub `nrs2013`・コード未経験）

---

## 1. 目的（何をするアプリか）

コンサートの LED 画面に映る「電飾」（線・電球・任意の図形）を、実際の **DMX 卓**（Art-Net 経由）で点灯・消灯・調色するためのツール。
電飾以外の部分は黒（＝透明扱い）で出力し、**Syphon** で **Resolume Arena** へ渡して、加算合成で本番映像に重ねて LED へ送る。

要するに「映像に描いた電飾を、卓の番地（DMX チャンネル）で RGB ＋バーチャルディマー操作して光らせる」。

## 2. 使う人（ペルソナ）

コンサート演出家。コード未経験。電飾の専門家ではないが、専門家の仲間に相談できる。
→ **パワーポイント感覚で図形を描ける手軽さ**が最重要。専門用語は出さず、迷ったら聞ける作りにする。

## 3. 中心の体験（コアフロー）

1. その現場の **チャート画像**（実際の LED 映像 / 写真）を読み込み、背景（作業台）に敷く。
2. その上に、パワポ感覚で電飾を描く（線・電球・三角など任意の図形、任意の位置・大きさ・形）。
3. 各図形に **開始番地（DMX アドレス）** と **構成（R/G/B/ディマー）** を割り当てる。
4. 本番モードで卓を動かすと、該当図形がその色・明るさで光る。
5. Syphon ソース「DECOR STUDIO」を Resolume が受け取り、**加算（Add）** で重ねて LED へ。

## 4. 全体構成（アーキテクチャ）

- **単一の Mac デスクトップアプリ（Electron）**。`.app` アイコンから起動（LED CHART PACKER と同じ運用感）。
- **同一 Mac 内**で Resolume と併用する前提（Syphon は同一 Mac 内でのテクスチャ受け渡し）。
- 内部の 3 系統：
  - **受信:** Art-Net（UDP 6454）を Node 側で待ち受け。net / subnet / universe に対応、複数ユニバース可。
  - **編集＆描画:** Chromium 上のキャンバス（SVG / Canvas / WebGL）。
  - **出力:** `node-syphon` で「DECOR STUDIO」ソースを発行（黒背景、必要に応じてアルファ付き）。
- **フォールバック:** `node-syphon` がビルド/動作不可な環境では、(a) NDI 送出、(b) 黒背景フルスクリーン窓を Resolume でキャプチャ、のいずれかに切替。
- **別 PC 構成（Resolume が別 Mac/PC）** になった場合は Syphon ではなく **NDI**（ネットワーク経由）へ差し替え。初版は同一 Mac・Syphon を本線とする。

## 5. 2 つのモード

- **編集モード:** 描く / 番地割当 / 画面サイズ設定 / 保存・読込 / 手動テスト。
- **本番モード:** 受信値で発光 / Syphon 出力 / 全画面プレビュー。

## 6. エディター仕様（描く）

- **キャンバス:** サイズ自由設定（新規の既定 1920×1080、上限の目安 4096×2160 程度＝4K 超えは想定外）。背景色は純黒。
- **背景下絵（作業台）:** 画像（PNG / JPG）を読み込み、不透明度を調整して敷く。位置合わせ専用で、**出力には含めない**。
- **図形ツール:** 直線 / 折れ線 / フリーハンド（ペン）/ 円・楕円（電球）/ 三角 / 四角 / 星 / 多角形。
- **各図形のプロパティ:**
  - 表示モード: **線（stroke）/ 塗り（fill）/ 両方** を選択。
  - 線の太さ、**グロー（ぼかし）半径・強度**、サイズ、回転、位置。
  - 番地割当（第 7 章）。
- **操作:** 選択・移動・リサイズ・回転・複製・削除・重ね順・整列・グリッド/スナップ（任意）。
- **一斉点灯:** 同一番地を複数図形に割当可（卓 1 ロで複数図形が同時に光る）。別番地なら個別制御。

## 7. DMX パッチ仕様（番地の付け方）

- **1 図形＝1 灯体（fixture）。** 開始番地（1..512）＋ユニバース指定を持つ。
- **既定チャンネル構成:** R, G, B（**3ch**）。例: 線① = 1〜3 番（1:R, 2:G, 3:B）。
  **バーチャルディマー（明るさ）は卓側で処理**し、RGB に掛けてから送る。アプリ側はディマー用の番地を消費しない。
- **選べる構成:**
  - **RGB（3ch、既定）** … 明るさは卓のバーチャルディマーで RGB に織り込み済み
  - RGB + Dimmer（4ch）… 卓にバーチャルディマーが無く、アプリ側で `RGB × Dim` を計算したい場合のみ
  - Dimmer のみ（1ch、色は固定値）
  - RGBW（5ch: R,G,B,W,Dimmer）※将来
- **発光色の計算:**
  - 既定（3ch）: `outRGB = 受信 RGB`（そのまま表示。明るさは卓で済んでいる）
  - 4ch を選んだ場合のみ: `outRGB = RGB × (Dimmer / 255)`
- **明るさカーブ:** ディマーは知覚的に自然なカーブ（sRGB 近似ガンマ）を任意で ON にできる。
- **重複チェック:** 同一ユニバース内でチャンネル範囲が部分的に重なる割当を警告。
  （同一番地の完全共有は「一斉点灯」の意図なので許可。半端な重なりだけ警告。）
- **パッチ一覧:** 図形名・ユニバース・開始番地・構成・現在値を表で表示。CSV / 印刷出力。
- **ユニバース管理:** 512 番を超えたら新ユニバース。受信側ユニバース番号と対応付け。

## 8. 受信仕様（Art-Net）

- UDP 6454 を待ち受け、ArtDMX パケットを解釈。Universe（15bit: net / subnet / universe）をパッチのユニバースに対応付け。
- **受信状態表示:** ユニバースごとに「来てる / 来てない」「最終受信時刻」をランプ表示。
- **取りこぼし対策:** 最新値を保持。一定時間未受信のとき「最後の値ホールド」か「ゼロに落とす」を設定で選択。
- **ネットワーク I/F 選択:** 受信する LAN ポート（複数 NIC 環境向け）を選べる。

## 9. 本番出力 / 描画

- 各図形を現在の `outRGB` で描画。stroke モードは輪郭が光り、fill モードは面が光る。グローはぼかしで表現。
- 背景は純黒（#000）。下絵は出力に含めない。
- **Syphon ソース名:**「DECOR STUDIO」（変更可）。出力サイズ＝キャンバスサイズ。
- **Resolume 運用:** 加算（Add）で重ねるのが既定。アルファ付き出力も用意し、通常合成も選べる。
- **全画面プレビュー窓:** 別ディスプレイに出して実寸確認できる。

## 10. テスト機能（卓なしでも確認）

- **手動フェーダー:** 各灯体の R / G / B / Dimmer をアプリ内スライダーで手動操作。卓が無くても点灯確認。
- **テストパターン:** 全点灯 / 順次点灯（チェイス）/ 番地確認（1 灯ずつ番号を表示）。

## 11. 保存 / ファイル（チャート管理）

- **1 チャート =** `{ キャンバスサイズ, 下絵, 図形配列, パッチ, ユニバース設定, Syphon 設定 }` を **JSON 1 ファイル**で保存。
- 下絵画像はファイル内に **埋め込み（base64）** を既定（持ち運びやすさ優先）。隣接ファイル保存も選択可。
- 新規 / 複製 / 読込 / 書き出し。最近使ったチャート一覧。
- 毎現場ごとに違うチャートを手早く作れることを重視（自由なサイズ・新規作成）。

## 12. 起動 / 運用

- `.app` アイコンで起動。
- 初回: Art-Net 受信ネットワーク I/F の選択、Syphon 有効化。
- **常時表示ランプ:** Art-Net 受信ランプ、Syphon 出力ランプ。

## 13. エラー処理

- **Art-Net 未受信:** ランプ赤。本番は「最後の値ホールド」（設定で「ゼロ」）。
- **ポート 6454 競合:** 警告と対処手順を表示。
- **Syphon 不可:** フォールバック手順（NDI / フルスクリーン窓キャプチャ）を案内。
- **ユニバース不一致:** パッチ側で受信ユニバースを再マップ可能。
- **番地重複:** 保存前に警告一覧を提示。

## 14. テスト戦略

- **単体:** チャンネル計算（RGB×Dimmer・ガンマ）、ArtDMX パース、パッチ重複検出、ファイル保存/読込の往復。
- **結合:** 擬似 Art-Net 送出（テスト送信機 / QLC+ など）→ 該当図形が点灯することを確認。
- **受け入れ:** 手動フェーダーで全灯体点灯 → Resolume で加算合成し黒が消えることを確認。実寸プレビューで見え方を確認。

## 15. やらないこと（初版の YAGNI）

- **ピクセルマッピング**（1 本の線を多数の番地で部分制御）→ 将来。初版は **1 図形＝1 色（均一）**。
- RGBW 以上の複雑な灯体プロファイル → 初版は RGB+Dimmer まで（RGBW は将来）。
- 音 / タイムコード連動・キュー記録 → 範囲外（卓側で操作する）。
- マルチ PC（NDI 本線）→ 初版は Syphon・同一 Mac。NDI はフォールバック設計のみ。

## 16. 名称

仮称「DECOR STUDIO」。確定でなければ後で変更可（`* STUDIO` シリーズに合わせている）。

## 用語ミニ辞典（迷ったら）

- **DMX / 番地（アドレス）:** 照明卓が灯体を操作する番号。1 ユニバース＝1〜512 番。
- **ユニバース:** 512 番ひとまとまり。足りなければ 2 個目・3 個目…と増やす。
- **Art-Net:** DMX を LAN ケーブル（ネットワーク）で流す方式。卓とパソコンを LAN で直結できる。
- **バーチャルディマー:** RGB の灯体に「全体の明るさ」フェーダーを与える**卓側の機能**。卓が明るさを RGB に掛けてから送るので、**DMX の番地は使わない**（だから 1 図形＝R/G/B の 3 本）。
- **Syphon:** 同じ Mac 内で、映像をアプリ間で直接渡す Mac 標準の仕組み（ケーブル不要・ほぼ遅延ゼロ）。
- **加算（Add）合成:** 映像を「足し算」で重ねる方法。黒い部分は足しても変わらない＝消える。だから電飾以外の黒が透明扱いになる。
